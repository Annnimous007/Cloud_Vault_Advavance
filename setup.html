Setup Guide

Get your personal CloudVault live on GitHub Pages in under 10 minutes.

1
Fork or Clone the Repository

Start by getting your own copy of CloudVault on GitHub.

  1. Go to the CloudVault GitHub repository.
  2. Click Fork to create a copy under your account — or clone it and push to a new repo.
  3. Name it something memorable, e.g. my-cloudvault or files.

You can use either a public or private repository. Public repos mean your files are accessible to anyone with the link (great for sharing). Private repos require a token even for reading.

2
Enable GitHub Pages

Turn on GitHub Pages to serve your CloudVault as a website.

  1. Open your repository on GitHub.
  2. Go to Settings → Pages.
  3. Under Source, select Deploy from a branch.
  4. Choose branch main, folder / (root).
  5. Click Save.

Your site will be live at:

https://YOUR_USERNAME.github.io/YOUR_REPO/

GitHub Pages is completely free for public repositories and also available on private repos with GitHub Pro, Team, or Enterprise plans.

3
Generate a GitHub Personal Access Token

A token lets CloudVault upload files to your repo via the GitHub API.

  1. Go to GitHub → Settings → Developer settings → Personal access tokens.
  2. Click Generate new token (classic).
  3. Give it a name like cloudvault-upload.
  4. Set expiration to your comfort (90 days or No expiration).
  5. Under scopes, check repo (full control of private repos) or public_repo for public repos.
  6. Click Generate token and copy it immediately — you won't see it again!

Security note: Never commit your token to a public repository. For production use, consider a serverless backend (Cloudflare Worker, Vercel Edge Function) that holds the token server-side and proxies uploads.

4
Edit js/config.js

Open js/config.js in your repo and fill in your details. This is the only file you need to edit.

// js/config.js
const CONFIG = {
  GITHUB_OWNER:  'your-github-username',
  GITHUB_REPO:   'your-repo-name',
  GITHUB_BRANCH: 'main',
  GITHUB_TOKEN:  'ghp_xxxxxxxxxxxxxxxxxxxx', // your token

  DEMO_MODE: false,  // ← Set this to false!

  // Optional: set if using a custom domain
  BASE_URL: '',
};

Make sure to set DEMO_MODE: false — otherwise CloudVault will keep showing the sample files instead of your real files.

5
Add Files to Your Repository

There are two ways to add files:

Option A — Upload via CloudVault UI (recommended)

  • Click the Upload button in the top bar.
  • Drag & drop files or browse your computer.
  • Select a category or let CloudVault auto-detect it.
  • Click Start Upload — files go directly to your GitHub repo!

Option B — Commit directly to GitHub

  • Place files in the uploads/ folder in your repo.
  • Use the correct subfolder: documents/, images/, videos/, audio/, archives/, or others/.
  • Commit and push — CloudVault picks them up automatically on next load.
6

7
Custom Domain (Optional)
  1. In your repo, go to Settings → Pages → Custom domain.
  2. Enter your domain, e.g. files.yourdomain.com.
  3. Add a CNAME DNS record at your registrar pointing to YOUR_USERNAME.github.io.
  4. Update BASE_URL in js/config.js to 'https://files.yourdomain.com'.

You're all set! Visit your GitHub Pages URL to see CloudVault live with your real files. Share any file by clicking the Copy Link or QR Code button — no login required for viewers.
